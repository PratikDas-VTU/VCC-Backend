Rules & Guidelines

Please read carefully and follow all rules

01

GitHub Repository Policy

A new GitHub repository must be created only after entering the college campus. All development work must be pushed to the repository strictly during the competition period.

02

Pre-written Code Prohibition

Pre-written or previously developed projects are strictly prohibited. Only code written during the event duration will be considered for evaluation.

03

Competition Time Window

The competition will run for a fixed development duration. Coding must begin only after the official start announcement by the organizers.

04

Final Submission Requirements

The final submission must be a working AI-based web application or web dashboard that runs either locally or directly in a web browser.

05

Technology Stack Freedom

Teams are free to use any web technology stack including HTML, CSS, JavaScript, React, Flask, Node.js, or similar technologies to implement their solution.

06

Fair Play & Collaboration Policy

Cross-team collaboration, code sharing, or idea exchange between teams is strictly prohibited and will result in disqualification.

07

Innovation & Conduct

Teams are encouraged to focus on innovation, creativity, and practical problem-solving rather than purely theoretical solutions. Any form of malpractice or rule violation will lead to immediate disqualification, and the judges‚Äô decision will be final.
