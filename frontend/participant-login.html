Rules & Guidelines

Please read carefully and follow all rules

01

GitHub Repository Policy

A new GitHub repository must be created only after entering the college campus. All development work must be pushed strictly during the competition period.
02

Pre-written Code Prohibition

Pre-written or previously developed projects are strictly prohibited. Only code written during the event will be considered for evaluation.
03

Competition Time Window

The competition will run for a fixed development duration. Coding must begin only after the official start announcement.
04

Final Submission Requirements

The final submission must be a working AI-based web application or web dashboard that runs locally or directly in a browser.
05

Technology Stack Freedom

Teams are free to use any web technology stack such as HTML, CSS, JavaScript, React, Flask, Node.js, or similar frameworks to implement their solution.
06

Fair Play & Collaboration

Cross-team collaboration, code sharing, or idea exchange between teams is strictly prohibited and will result in disqualification.
07

Innovation & Creativity

Teams are encouraged to demonstrate innovation, creativity, and practical problem-solving rather than focusing only on theoretical concepts.
08

Malpractice & Disqualification

Any form of malpractice or violation of rules will lead to immediate disqualification. The judges‚Äô decision will be final and binding.
